# design-sync notes — KiwiVoca Design System

App-shaped repo (`kiwivoca-frontend`, private Vite app), synced as the **package shape** in **synth-entry mode**: no built dist, so the bundle is built from the `ui/` barrel `frontend/src/components/ui/index.ts` (`cfg.entry`) and the 15 components are pinned via `cfg.componentSrcMap`. Target project: **KiwiVoca Design System** (`eaf9ddc4-96f0-46d1-a789-bfbd3f1bf6a6`).

## Build pipeline
- **CSS is compiled, not shipped.** The app styles with Tailwind utilities + CSS-variable tokens; there is no compiled stylesheet in the repo. `cfg.buildCmd` = `node .design-sync/build-styles.mjs` compiles `frontend/src/index.css` (tokens + `@tailwind` + custom utilities) against the app config, prepends the brand-font CDN `@import`s, and writes `frontend/.ds-styles.css` (= `cfg.cssEntry`). **Always run buildCmd before package-build** (the resync driver does this automatically; a bare `package-build` does not).
- The build-styles content glob includes `../.design-sync/previews/**` so utility classes used only in previews aren't purged. If a preview uses a brand-new utility class and renders unstyled, re-run buildCmd then rebuild.
- Brand fonts (Pretendard, Quicksand, Noto Sans JP) load via remote `@import` → validate prints `[FONT_REMOTE]` (informational, expected, no action).

## i18n provider
- `Sheet`, `ConfirmSheet`, `SpeakButton` call react-i18next `useTranslation`. Without an instance their visible labels render as raw keys (e.g. ConfirmSheet's Cancel button). `.design-sync/preview-providers.tsx` initialises i18next with the app's real **ko** locale (`common` namespace) and is wired via `cfg.provider` + `cfg.extraEntries`, so all cards show genuine Korean labels.

## Overlay components (Sheet, ConfirmSheet, ToastProvider)
- They render `position: fixed inset-0`, which contributes nothing to layout → capture crops to a sliver. **Wrap the preview in a `Stage` div with `transform: translateZ(0)`** (makes it the containing block for the fixed descendant) + `position: relative`, `overflow: hidden`, explicit width/height.
- **Capture height ≈ the override viewport height** and bottom-sheets are `items-end`. Set `cfg.overrides.<Name>.viewport` and make the Stage height **a bit LESS** than that viewport height, or the bottom-aligned sheet clips/escapes. ConfirmSheet: override `440x440`, Stage `420x420` → full sheet visible.

## Preview authoring conventions (validated across all 15)
- **Inline `style={{}}` for layout wrappers** (rows, fixed widths, columns) — NOT Tailwind utility classes, which can be purged from the compiled CSS when used only in previews. Components bring their own token styling. (The solo Button/TextField previews use Tailwind wrappers and still render because build-styles globs `previews/**` at full-build time; inline styles are the robust default regardless.)
- Import components from `kiwivoca-frontend`; icons from `lucide-react`. All `ui/` components are default exports re-exported via the barrel.
- **SegmentedControl** is controlled — pass fixed `value` + `onChange={()=>{}}` and a UNIQUE `layoutId` per instance, or framer-motion's shared-layout pill animates across cells.
- **EmptyState** renders the KiwiBuddy mascot per `mood`; **SpeakButton** renders (supported is true in the capture browser).

## ToastProvider capture (timing — SOLVED)
- The toast stack enters with a framer-motion **opacity** animation. package-capture does a bare goto (warms the CDN-font cache) then a `?story=` goto; with fonts cached the second navigation hits networkidle instantly and screenshots while opacity is still 0 → blank card. `reducedMotion="always"` does NOT help (it disables transform/layout, not opacity).
- **Fix (in `previews/ToastProvider.tsx`):** inject `<style>{`[role="status"]{opacity:1 !important;transform:none !important;}`}</style>` to force the settled visual state (beats framer-motion's inline opacity). Toasts fire once on mount via `useToast()` inside the Stage. Verified under the real double-goto flow (opacity 1/1/1). Stage 420×220 under the `440x320` override.

## Overlay viewport heights that worked
- ConfirmSheet: Stage 420×420 under override `440x440`.
- Sheet: Stage 420×500 under override `440x560`.
- ToastProvider: Stage 420×300 under override `440x320`.

## Known render warns
- `[FONT_REMOTE]` on Pretendard/Quicksand/Noto Sans JP — expected (CDN fonts).

## Re-sync risks
- `cfg.cssEntry` (`frontend/.ds-styles.css`) and `frontend/.ds-input.css` are generated by buildCmd and gitignored — never hand-edit; re-run buildCmd.
- The bundle inlines `framer-motion` + `lucide-react` from source (no dist), so a major bump to those could change render output — re-grade after such bumps.
- ToastProvider preview shows toasts triggered on mount; toasts auto-dismiss after 3.2s — if its card ever captures blank, it's a capture-timing race (re-capture).
